GitHub Pages vs Netlify

Migration assessment para Propel ROI

GitHub Pages (actual)

  • ✗ SPA routing roto al refrescar (necesita hack 404.html)
  • ✗ Sin deploy previews por PR
  • ✗ Sin redirects server-side
  • ✗ Base path obligatorio: /propel-roi/
  • ✗ Env vars solo via GitHub Secrets
  • ✗ Sin rollback instantaneo
  • ✓ Gratis ilimitado
  • ✓ Zero config si ya usas GitHub

Netlify

  • ✓ SPA routing nativo (rewrites)
  • ✓ Deploy preview por cada PR
  • ✓ Redirects, headers, rewrites
  • ✓ Custom domain + auto SSL
  • ✓ Env vars en UI (no secrets)
  • ✓ Rollback en 1 click
  • ✓ Branch deploys automaticos
  • ⚠ Free tier con limites (ver abajo)

Veredicto: migrar

El pain principal es el SPA routing roto en GitHub Pages (refresh = 404) y el base path /propel-roi/ que complica assets y links. Netlify resuelve ambos con una linea de config. Los limites del free tier no son problema para una herramienta interna.

Riesgo: zero. Es un SPA estatico con Supabase backend. No hay server-side rendering ni vendor lock-in. Si Netlify no convence, volver a GitHub Pages es revertir un commit.

Roadmap de migracion

1

Crear netlify.toml

Config de build + SPA rewrite. Un archivo en la raiz del repo:

[build]
  command = "npm run build"
  publish = "dist"

[[redirects]]
  from = "/*"
  to = "/index.html"
  status = 200

[build.environment]
  NODE_VERSION = "20"
2 min
2

Quitar VITE_BASE_PATH

Ya no hace falta /propel-roi/ como base path. Netlify sirve desde la raiz /.

Eliminar de vite.config.ts el base: process.env.VITE_BASE_PATH o ponerlo a "/".

2 min
3

Conectar repo en Netlify

Netlify UI: New site > Import from GitHub > lucassiroo-eng/propel-roi. Detecta el netlify.toml automaticamente.

3 min
4

Env vars en Netlify

En Site settings > Environment variables, anadir:

VITE_SUPABASE_URL = https://dsmovbahvtbzxgfyndyj.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY = eyJ...
VITE_SUPABASE_PROJECT_ID = dsmovbahvtbzxgfyndyj
2 min
5

Deploy + verificar

Push a main > Netlify build automatico > URL tipo propel-roi.netlify.app. Verificar que el routing funciona al refrescar.

5 min
6

Custom domain (opcional)

Si quieres roi.factorial.tools o similar: Netlify UI > Domain settings > Add custom domain. Auto SSL incluido.

10 min
7

Desactivar GitHub Pages

Repo settings > Pages > desactivar. Borrar .github/workflows/deploy.yml (mantener el deploy-functions job en un workflow separado si quieres auto-deploy de edge functions).

2 min

Total estimado: ~25 minutos, zero downtime
